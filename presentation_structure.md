## Слайд 7: Теоретическая база — LoRA

**Секция внизу:** МЕТОДОЛОГИЯ
**Подсекция сверху:** Метод дообучения

**Оформление:** Схема метода LoRA + ключевые формулы (как в референсе — визуальная схема + текст справа).

**Слева — схема LoRA:**
Визуальная диаграмма: замороженная матрица W₀ + низкоранговое обновление BA

```
          W₀ (frozen)
Input x ──────────────── + ──→ Output
          ↘              ↗
           A (r×d) → B (d×r)
           (trainable)
```

**Справа — ключевые свойства:**
- Формула: W = W₀ + BA, где rank(BA) = r ≪ d
- Обучается < 1% параметров
- Сохраняет качество полного fine-tuning
- Идеально для коммерческих LLM через API

**Вывод:** LoRA позволяет адаптировать коммерческую модель (YandexGPT) без доступа к полным весам, обучая лишь компактные матрицы обновлений.

| train\_synth\_* | 8–24 | 1 | GEO массово как TAG |
| train\_real\_250 | 1 | 21 | TAG→GEO (дисбаланс) |
| **train\_mixed\_500** | **0** | **1** | **Практически нет** |

**Визуальная схема (3 блока):**
```
Baseline:        Синтетика:        Микс:
TAG ════► GEO    GEO ══════► TAG   ✓ Баланс
(57 ошибок)      (8-24 ошибок)     (1 ошибка)
Bias →           ← Инверсия bias   Идеальная граница
```

**Вывод:** Синтетика перекомпенсирует bias (инверсия). Микс данных находит оптимальную решающую границу между классами.
